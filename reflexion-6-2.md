1. Una rama de Git es como una linea de tiempo alternativa. Nos permite hacer cambios a nuestro proyecto sin afectar la rama principal.

2. Nos permite hacer cambios que podrían afectar el funcionamiento del proyecto en un entorno aislado, para evitar que nuestros cambios 
tengan repecrusiones.

3. Conflictos frecuentes con push/pull, pérdida de trabajo si alguien reescribe nuestros cambios, dificultad para saber el originador de un 
cambio, mayor riesgo de añadir bugs al programa.
